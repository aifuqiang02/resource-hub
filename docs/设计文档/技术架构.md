# 技术架构

## 1、整体架构

采用前后端分离架构，前端使用 Next.js + React，后端使用 Node.js + Python。

## 2、系统架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (Next.js)                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐   │
│  │   首页/搜索  │  │ AI 对话助手  │  │  用户中心       │   │
│  └─────────────┘  └─────────────┘  └─────────────────┘   │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP / WebSocket
┌──────────────────────────▼──────────────────────────────────┐
│                      API Gateway (Nginx)                    │
│                     负载均衡 / 限流 / 路由                     │
└──────────────────────────┬──────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
┌────▼────┐        ┌──────▼──────┐      ┌──────▼──────┐
```

## 3、技术选型

### 前端

| 技术 | 用途 |
|------|------|
| Next.js | React 框架 |
| TypeScript | 类型安全 |
| Tailwind CSS | 样式方案 |
| Zustand | 状态管理 |
| Axios | HTTP 客户端 |

### 后端

| 技术 | 用途 |
|------|------|
| Node.js | API 服务 |
| Python | AI 推理服务 |
| FastAPI | Python Web 框架 |
| Prisma | ORM |

### 数据层

| 技术 | 用途 |
|------|------|
| PostgreSQL | 主数据库 |
| Redis | 缓存、会话 |
| Elasticsearch | 搜索引擎 |

### 基础设施

| 技术 | 用途 |
|------|------|
| Docker | 容器化 |
| Nginx | 反向代理 |
| PM2 | 进程管理 |

## 4、模块设计

### 4.1 Web API 模块

- 用户认证（注册、登录、JWT）
- 资源管理（CRUD）
- 评价管理（评分、评论）
- 收藏管理
- 下载链接生成

### 4.2 AI 服务模块

- **当前**: 集成 OpenCode 对话能力
- **未来**: 作为公共服务供其他项目调用

#### AI 服务职责

| 功能 | 说明 |
|------|------|
| 意图识别 | 理解用户搜索意图 |
| 资源推荐 | 根据描述推荐相关资源 |
| 对话管理 | 支持多轮对话上下文 |

#### AI 服务接口

```python
# AI 推荐服务
async def recommend_resources(user_query: str, context: list) -> list:
    # 1. 理解用户需求
    # 2. 检索相关资源
    # 3. 生成推荐结果
    return recommendations
```

### 4.3 搜索服务模块

- 关键词索引
- 分词、同义词处理
- 相关度排序
- 热门搜索词缓存

### 4.4 AI 公共服务（未来）

- 独立部署的 AI 对话服务
- 提供 SDK 供各项目集成
- 支持自定义提示词
- 支持知识库配置

## 6、扩展性设计

- **水平扩展**: API、AI、Search 服务均可多实例部署
- **缓存策略**: Redis 缓存热点数据
- **异步任务**: 下载统计、推荐更新使用消息队列
- **AI 公共服务**: 预留接口，支持后续多项目集成
